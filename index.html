How is the data encrypted?

What happens with the data?

Encrypti0n.com's algorithms are secure by default. Your data gets encrypted with an AES-GCM algorithm with a securely derived 256-bit key directly using the web crypto API on your device.

The 256-bit key gets derived using your input password and the PBKDF2 key derivation algorithm.

What is the PBKDF2 key derivation algorithm?

What happens with the password?

PBKDF2 is explicitly designed for password-based key derivation. It mitigates the risks of using a weak password by introducing computational cost and salting to resist brute-force and rainbow table attacks.

You can customize the rounds and salt by opening the PBKDF2 Options (if you know what you are doing).

By default, the site uses a random, secure 16-byte salt and 5M SHA-256 rounds on every action.

What is AES-GCM Encryption?

What is the output?

AES-GCM 256-bit encryption is an advanced cryptographic method combining the AES algorithm with GCM mode, which provides both data encryption and integrity verification.

AES-GCM is secure because it uses AES in counter mode to encrypt data, ensuring confidentiality. It also generates a Galois authentication tag to verify integrity and detect tampering.

By default, the site generates a new random, secure 12-byte nonce on every action.

Encrypt Application Data

Encrypts all locally saved data (e.g. passwords, names, configs etc.) with a master password.

The encryption process uses AES-GCM and PBKDF2 with a random salt and 5M rounds. Don't close the windows while encrypting.

Attention: If you forget your master password, you can't decrypt your app data anymore.

The two entered passwords don't match.

You have to enter your secure master password two times to encrypt the applications data.
